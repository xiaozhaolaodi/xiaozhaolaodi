webpack 学习记录介绍

一、webpack 简介

1. webpack 是什么？

  • webpack 是一种前端资源构建工具，是一个静态模块打包器（module bundler）;

  • 在webpack看来，前端的所有静态资源文件（js/json/css/img/less/……）都会被作为模块进行处理;

  • 根据各个模块之间的依赖关系进行静态分析，打包生成对应的静态资源(bundle);

2. webpack 的五个核心概念

 2.1 Entry

  入口（Entry）：指示webpack以哪个文件为入口或起点开始打包以及分析构建内部依赖图;

 2.2 Output

  输出（Output）：指示webpack打包后的资源bundles输出到哪里去，以及如何命名;

 2.3 Loader

  Loader：让webpack能够去处理那些非JavaScript文件（webpack自身只能处理JavaScript）

 2.4 Plugins

  插件（Plugins）：可以用于执行范围更广的任务，插件的范围包括，从打包优化和压缩，一直到重新定义环境中的变量等;

 2.5 Mode 模式

  模式（Mode）：指示webpack使用相应模式的配置，通过选择 development 或 production 之中的一个，来设置 mode 参数，你可以启用相应模式下的 webpack 内置的优化;

选项 描述 特点
development 会将 process.env.NODE_ENV 的值设为 development
启用 NamedChunksPlugin 和 NamedModulesPlugin		 开发环境，也就是让开发者可以
对代码进行本地调试运行的环境
production 会将 process.env.NODE_ENV 的值设为 production
启用 FlagDependencyUsagePlugin, FlagIncludedChunksPlugin,
ModuleConcatenationPlugin, NoEmitOnErrorsPlugin,
OccurrenceOrderPlugin, SideEffectsFlagPlugin 和 UglifyJsPlugin		 生产环境可以让代码上线运行的环境

二、webpack 初体验

1. 初始化配置

 1.1 初始化 package.json

  输入指令：npm init

 1.2 下载并安装 webpack

  全局安装 webpack 输入指令：npm install webpack webpack-cli -g

  本地安装 webpack 输入指令：npm install webpack webpack-cli -D

2. 打包应用测试

 2.1 开发环境打包

  • webpack 以 ./src/index.js 为入口文件开始打包，打包后输出到 ./build/built.js

  • 开发环境打包指令：webpack ./src/index.js -o ./build/built.js --mode=development

 2.2 生产环境打包
  • 生产环境打包指令：webpack ./src/index.js -o ./build/built.js --mode=production
 2.3 结论
  • webpack可以直接处理js和json资源，不能处理css/img等其他资源;
  • 生产环境和开发环境可以将ES6的模块化语法转换成浏览器识别的模块化语法
  • 生产环境相比于开发环境多了一个压缩js代码的功能

  • webpack.config.js
    const { resolve } = require('path');
const HtmlWebpcakPlugin = require('html-webpack-plugin');
module.exports = {
  mode: 'development',
  entry: './src/index.js',
  output: {
      filename: 'built.js',
      path: resolve(__dirname, 'build')
  },
  module: {
      rules: [
          {
              // 打包css资源的loader配置
              // 需要使用多个loader处理的话使用 use

              use: ['style-loader', 'css-loader', 'less-loader']
          },
          {
              // 打包图片资源的loader配置 需要用到url-loader
              // 使用单个loader处理使用 loader
              test: /\.(jpg|png|gif)$/,
              // 需要下载两个包 url-loader file-loader
              // 问题：这个loader默认情况下是不能处理html中的图片
              loader: 'url-loader',
              options: {
                  // 当图片大小小于8kb的话就会被转化成base64进行处理
                  // 优点：减少请求数量，减轻服务器压力
                  // 缺点：图片体积会更大（文件请求速度更慢一点）
                  limit: 8 * 1024,
                  // 给打包后的文件命名：hash的前十位以及原文件的扩展名
                  name: '[hash:10].[ext]'
              }
          },
          {
              test: /\.html$/,
              // 用来处理html文件中的img资源的（负责引入图片从而被url-loader进行处理）
              loader: 'html-loader'
          }
      ]
  },
  plugins: [
      new HtmlWebpcakPlugin({
          template: './src/index.html'
      })
  ]
}

5. 打包其他资源

  • 这里的其他资源是指除了之前打包的资源之外的其他资源；
  • 使用的loader是 file-loader；
  • 只需要在 webpack.config.js 的module配置中添加下面代码：
    {
  // exclude 表示排除匹配的文件类型
  exclude: /\.(js|css|html|less)/,
  loader: 'file-loader',
  options: {
      name: '[hash:10].[ext]'
  }
}

6. devServer 配置

  • 在我们实际开发过程中，我们是在一直修改src目录下的文件，目前我们学习的情况下，如果修改代码之后在浏览器查看效果的话，需要每次都执行 webpack 命令；

  • 解决这个问题我们需要使用开发服务器 devServer ,用来实现自动化,启动之后可以监控代码的变化，进而进行自动编译，自动刷新浏览器，集大程度的提高了我们的开发效率；

  • 安装 webpack-dev-server, 执行：npm install webpack-dev-server -D

  • 在 webpack.config.js 的 module.exports 配置中添加下面配置：

    // 开发服务器是运行在内存中的，因此每次更新编译的时候不会对output位置的东西进行修改
devServer: {
  // 项目构建后的路径
  contentBase: resolve(__dirname, 'build'),
  // 启用gzip压缩
  compress: true,
  // 端口号
  port: 3000,
  // 启动后打开浏览器
  open: true
}

  • 运行命令：npx webpack-dev-server

四、webpack 生产环境的基本配置

1. 以上开发环境打包存在的一些问题

 1.1 打包之后可能会出现闪屏现象：
  • 因为打包时通过css-loader之后是将css添加在js文件中，又经过style-loader创建style标签加入样式，当整个过程比较慢的时候页面会出现闪屏现象；
  • 安装插件：npm install mini-css-extract-plugin;
  • 配置文件做如下配置：(打包后将css文件提取出来通过link的方式引入页面)
    const { resolve } = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugins = require('mini-css-extract-plugin')
module.exports = {
  mode: 'development',
  entry: './src/index.js',
  output: {
      filename: 'index.js',
      path: resolve(__dirname, 'dist')
  },
  module: {
      rules: [
          {
              test: /\.css$/,
              use: [
                  // 作用：这个loader将css单独提取出来,通过link标签引入html
                  MiniCssExtractPlugins.loader,
                  // 将css文件整合到js中
                  'css-loader'
              ]
          }
      ]
  },
  plugins: [
      new HtmlWebpackPlugin({
          template: './src/index.html'
      }),
      new MiniCssExtractPlugins({
          // 对输出的css文件进行重命名
          filename: 'css/main.css'
      })
  ]
}

2. CSS 兼容性处理

  • 创建以下目录结构：

  • 安装 postcss-loader npm install postcss-loader -D

  • 安装 postcss-preset-env npm install postcss-preset-env -D

  • 配置文件做以下配置：

    const { resolve } = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugins = require('mini-css-extract-plugin');
// 设置nodejs的环境变量
process.env.NODE_ENV = 'development';
module.exports = {
  mode: 'development',
  entry: './src/index.js',
  output: {
      filename: 'js/index.js',
      path: resolve(__dirname, 'dist')
  },
  module: {
      rules: [
          {
              test: /\.css$/,
              use: [
                  MiniCssExtractPlugins.loader,
                  'css-loader',
                  /**
                   * 要处理css兼容性问题，必须要使用postcss：postcss-loader,
                   * 插件：postcss-preset-env 作用：帮助postcss在package.json中找到browserslist里面的配置，
                   *      通过配置加载指定的css兼容性样式
                   * "browserslist": {
                   *      // 开发环境配置 --> 需要设置node的环境变量：process.env.NODE_ENV = 'development'
                          "development": [
                              "last 1 chrome version",
                              "last 1 firefox version",
                              "last 1 safari version"
                          ],
                          // 生产环境配置： 默认情况下都是生产环境配置
                          "production": [
                              ">0.2%",
                              "not dead",
                              "not op_mini all"
                          ]
                      }
                   */
                  // 使用loader的默认配置
                  //'postcss-loader',
                  // 修改loader的配置
                  {
                      loader: 'postcss-loader',
                      options: {
                          postcssOptions: {
                              // ident: 'postcss',
                              plugins: [
                                  require('postcss-preset-env')()
                              ]
                          }
                      }
                  }
              ]
          }
      ]
  },
  plugins: [
      new HtmlWebpackPlugin({
          template: './src/index.html'
      }),
      new MiniCssExtractPlugins({
          filename: 'css/main.css'
      })
  ]
}

  • 修改package.json

  "development": [
      "last 1 chrome version",
      "last 1 firefox version",
      "last 1 safari version"
  ],
  "production": [
      ">0.2%",
      "not dead",
      "not op_mini all"
  ]
}

5. js兼容性处理

  • 下载安装包：
    npm install babel-loader@8.0.0-beta.0 @babel/core @babel/preset-env -D

  • 配置文件进行如下配置：

    const HtmlWebpackPlugin = require('html-webpack-plugin');
const { version } = require('os');
const { resolve } = require('path');
module.exports = {
  mode: 'development',
  entry: './src/js/index.js',
  output: {
      filename: 'js/index.js',
      path: resolve(__dirname, 'dist'),
  },
  module: {
      rules: [
          /**
           * js兼容性处理：安装 npm install babel-loader@8.0.0-beta.0 @babel/core @babel/preset-env -D
           * 1.基本的兼容性处理 --> @babel/preset-env
           *      问题：只能转换基本语法（如：promise）不能转换
           * 2.全部的js兼容性处理 --> @babel/polyfill3(目前不考虑)
           *      问题：我只需要解决部分兼容性问题，但是这个方法将锁头的兼容性代码全部引入，体积太大
           * 3.需要做兼容性处理的叫做：按需加载 --> core-js
           */
          {
              test: /\.js$/,
              exclude: /node_modules/,
              loader: 'babel-loader',
              options: {
                  // 预设：指示babel做怎样的兼容性处理
                  presets: [
                      [
                          '@babel/preset-env',
                          {
                              // 按需加载
                              useBuiltIns: 'usage',
                              //指定core-js的版本
                              corejs: {
                                  version: 3
                              },
                              // 兼容性做到哪个版本浏览器
                              targets: {
                                  chrome: '60',
                                  firefox: '60',
                                  ie: '9',
                                  safari: '10',
                                  edge: '17'
                              }
                          }
                      ]
                  ]
              }
          }
      ],
  },
  plugins: [
      new HtmlWebpackPlugin({
          template: './src/index.html',
      }),
  ],
};
