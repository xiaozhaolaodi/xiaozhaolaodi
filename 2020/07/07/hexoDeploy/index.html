hexo博客在云服务器上的部署

博客
发布日期:   2020-07-07
文章字数:   1k

今天来记录一下部署hexo的步骤

部署所需的系统配置

  • 本地windows 10系统；
  • 云服务器是阿里云 CentOS7 系统；
  • 如果想要使用域名的话可以准备一个域名；

本地主机所需配置

  • 安装 Git；
  • 安装 NodeJs；
  • 安装 Hexo；
  • 根据 Hexo官网 创建自己的博客，可以选择一个自己喜欢的主题；

云主机所需配置

  • 安装git，为了在云服务器上创建仓库；
  • 安装nginx，利用反向代理将请求代理到博客资源所在的位置；

接下来开始进行配置

1、 生成 ssh 公钥

  在本地windows进入win+R进入cmd(命令行终端)执行以下命令：

  ssh-keygen -t rsa

  如果有选择就同意，没有的话就一直点击回车；执行成功之后会在

我的电脑/C盘/用户/你自己的用户名/.shh/

  所在的位置生成 id_rsa，id_rsa.pub 文件，如果出现这些文件，证明成功。

注：如果后边遇到问题需要重新生成公钥的话，建议删除该文件夹下的所有文件重新生成！

2、 远程CentOS系统配置步骤

安装git，使用下面命令：

  yum install git

创建Git账户：

  addUser git

设置git账户密码

  sudo passwd git

按照提示输入两次密码

给账户添加权限：

  chmod 740 /etc/sudoers
  vim /etc/sudoers

找到以下配置：

## Allow root to run any commands anywhere
root    ALL=(ALL)     ALL

在后面添加：

git    ALL=(ALL)     ALL

保存退出并且改回权限

chmod 400 /etc/sudoers

配置云服务器上的 ssh

首先切换到git用户，创建 ~/.ssh 文件夹和 ~/.ssh/authorized_keys 文件，设置相应的权限

su git
mkdir ~/.ssh
vim ~/.ssh/authorized_keys
# 然后将win10中生成的.ssh文件夹下的 id_rsa.pub 文件中的公钥复制到authorized_keys
chmod 600 /home/git/.ssh/authorized_keys
chmod 700 /home/git/.ssh

完成之后需要在本地windows主机上测试是否可以免密登录git,在windows的命令行中使用：

#SERVER 代表云主机的ip地址
ssh -v git@SERVER

如果选择yes之后直接登陆成功的话说明配置成功。

在云服务器上配置 git 仓库

首先创建文件夹作为git仓库的目录并修改权限(使用root用户)

mkdir /var/repository
chown -R git:git /var/repository
chmod -R 755 /var/repository

然后创建网站的根目录(也就是博客文件的目录),x修改权限

mkdir /home/hexo
chown -R git:git /home/hexo
chmod -R 755 /home/hexo

然后创建一个 Git 裸仓库

cd /var/repository
git init --bare hexoBlog.git

创建一个新的 Git 钩子，用于自动部署

我们找到 /var/repository/hexoBlog.git 下自动生成的hooks文件夹，在里边新建post-receive文件

vim /var/repository/hexoBlog.git/hooks/post-receive

并且在里边配置如下命令：

#!/bin/bash
git --work-tree=/home/hexo --git-dir=/var/repository/hexoBlog.git checkout -f

将工作树( 项目地址 )和git仓库( hexoBlog.git )关联起来,这样从本机提交上的项目就可以同步到 /home/hexo 路径下，保存后修改文件权限，添加可执行权限：

chown -R git:git /var/repository/hexoBlog.git/hooks/post-receive
chmod +x /var/repository/hexoBlog.git/hooks/post-receive

在云服务器上配置 Nginx

安装nginx ( 大家到百度自行查询安装方法 )

修改nginx配置文件 nginx.conf 中的配置

server {
        listen       80;    #nginx反向代理的端口号
        server_name  【填写域名或者云主机IP地址】; 
        root       /home/hexo;       #网站的根目录
        location / {

        }
        error_page 404 /404.html;         #配置40x页面
        location = /40x.html {
        }
    }

修改完成之后重启nginx服务

配置本地hexo项目的配置文件 _config.yml

找到 deploy: 进行如下配置

deploy:
  type: git
  repo: root@Server:/var/repository/hexoBlog.git    
  branch: master      # 分支

** 注意：这里的格式一定要严格的按照缩进来进行 **

然后找到hexo项目目录 ， 在文件路径输入 cmd 进入命令行 执行下面命令：

hexo c
hexo g -d

如果执行结束之后显示成功，就可以在浏览器输入IP地址进行访问了
